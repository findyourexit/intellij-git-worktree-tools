# Support

Support for Git Arborist is provided on a best-effort basis through GitHub Issues.

## Before opening an issue

1. Check the [existing issues](https://github.com/findyourexit/intellij-git-arborist/issues) to see whether the problem has already been reported.
2. Collect the information listed below — reports without it are hard to act on.

## What to include in a bug report

- **IDE and version** — e.g. IntelliJ IDEA 2025.2.6 Ultimate, from **Help > About**.
- **Plugin version** — from **Settings > Plugins**, shown next to Git Arborist.
- **OS and architecture** — e.g. macOS 15.4 arm64, Windows 11 x64.
- **Git version** — output of `git --version`.
- **Steps to reproduce** — the exact sequence of actions that triggers the problem.
- **Expected behavior** — what you expected to happen.
- **Actual behavior** — what happened instead, including any error messages shown in the IDE.
- **IDE log** — if the IDE shows an exception or the action silently fails, attach the relevant portion of the log from **Help > Show Log in Finder/Explorer**.

Use the [Bug Report](https://github.com/findyourexit/intellij-git-arborist/issues/new?template=bug_report.yml) template to open a report.

## Feature requests

Open a [Feature Request](https://github.com/findyourexit/intellij-git-arborist/issues/new?template=feature_request.yml). Describe the problem you are trying to solve and the behavior you would like.

## Security issues

See [SECURITY.md](SECURITY.md). Do not open a public issue for security vulnerabilities.

## Out of scope

- Bugs in IntelliJ IDEA, Android Studio, or Git itself — report those upstream.
- Feature requests that require modifying IDE internals not exposed through public platform APIs.
- Support for IDE versions before build 252 (IntelliJ Platform 2025.2).
